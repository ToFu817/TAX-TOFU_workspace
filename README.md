## ✨ 功能特色

| 模組 | 說明 |
|------|------|
| 📊 儀表板 | 任務狀態總覽、待收款統計、客戶概況 |
| 📋 任務管理 | 建立/編輯/刪除任務，勾選完成，管理員審核 |
| 📝 任務項目 | 管理任務類型清單（營業稅申報、帳務處理等） |
| 👥 群組管理 | 員工資料、權限設定（管理員/使用者） |
| 🏢 客戶資料 | 完整客戶資料管理，支援搜尋篩選 |
| 🔀 客戶分配 | 將待分配客戶指派給承辦人 |
| ⚙️ 工作流程SOP | 建立標準作業流程，一鍵套用到工作任務 |
| 💰 收費資料 | 收款進度追蹤、各承辦業績統計 |
| 📅 年度日程 | 全年度例行工作日曆 |

### 🎯 特色功能

- **自動帶入**：輸入客戶編號自動填入公司名稱、承辦等資料
- **Excel 批次匯入**：所有模組都支援 Excel (.xlsx/.xls/.csv) 批次匯入
- **SOP 一鍵套用**：選擇 SOP + 客戶，自動建立包含所有流程步驟的任務
- **權限控管**：管理員可審核任務、查看收費統計；一般使用者僅能操作自己的任務
- **豆腐風 UI**：柔軟圓潤的視覺設計，搭配彈跳動畫回饋

## 🚀 安裝步驟

### 第一步：建立 Google Sheets

1. **開啟 Google Sheets**
   - 前往 [Google Sheets](https://sheets.google.com)
   - 點擊 **「+」建立空白試算表**
   - 將試算表命名為 **「案件管理系統」**

2. **記下試算表 ID**
   - 從網址列複製試算表 ID
   - 網址格式：`https://docs.google.com/spreadsheets/d/【這段就是ID】/edit`
   - 例如：`1a2B3c4D5e6F7g8H9i0J...`

> **💡 提示**：不需要手動建立工作表！下一步的 Apps Script 會自動幫你建立所有工作表和初始資料。

---

### 第二步：設定 Google Apps Script

1. **開啟 Apps Script 編輯器**
   - 在你剛建立的 Google Sheets 中
   - 點擊上方選單 **「擴充功能」→「Apps Script」**
   - 會開啟新的 Apps Script 編輯器頁面

2. **貼上後端程式碼**
   - 在編輯器中，刪除預設的 `function myFunction(){}` 
   - 將 `google-apps-script/Code.gs` 的**全部內容**複製貼上
   - 按 **Ctrl+S** 儲存

3. **初始化工作表**
   - 在 Apps Script 編輯器上方的函式選單中
   - 選擇 **`initializeSheets`**
   - 點擊 **▶️ 執行** 按鈕
   - 首次執行會要求授權：
     1. 點擊「查看權限」
     2. 選擇你的 Google 帳號
     3. 點擊「進階」→「前往（不安全）」
     4. 點擊「允許」
   - 執行完成後，回到 Google Sheets 確認以下工作表已自動建立：
     - ✅ 群組管理（含預設管理員帳號）
     - ✅ 客戶資料
     - ✅ 客戶分配
     - ✅ 工作任務
     - ✅ 任務項目（含預設項目）
     - ✅ 工作標準流程SOP（含工商登記流程）
     - ✅ 收費資料

4. **部署為 Web App**
   - 在 Apps Script 編輯器中
   - 點擊右上角 **「部署」→「新增部署作業」**
   - 左側齒輪圖示選擇 **「網頁應用程式」**
   - 設定：
     - **說明**：案件管理系統 API v1
     - **執行身分**：**我**
     - **存取權限**：**所有人**
   - 點擊 **「部署」**
   - 🔑 **複製產生的「網頁應用程式」URL**（格式如 `https://script.google.com/macros/s/AKfy.../exec`）


> **⚠️ 重要**：每次修改 Apps Script 程式碼後，必須重新部署才能生效：
> - 點擊「部署」→「管理部署作業」
> - 點擊鉛筆圖示編輯
> - 版本選「建立新版本」
> - 點擊「部署」

---

### 第三步：安裝前端

1. **安裝依賴套件**
   ```bash
   cd d:\acclaw_workspace
   npm install
   ```

2. **設定環境變數**
   - 編輯 `.env` 檔案
   - 將第二步複製的 Web App URL 貼上：
   ```env
   VITE_GAS_URL=https://script.google.com/macros/s/你的部署ID/exec
   ```

3. **啟動開發伺服器**
   ```bash
   npm run dev
   ```

4. **開啟瀏覽器**
   - 預設網址：`http://localhost:5173`
   - 使用預設帳號登入：
     - 管理員：帳號 `admin` / 密碼 `admin`
     - 一般使用者：帳號 `user` / 密碼 `user`

> **💡 開發模式**：如果 `.env` 中的 `VITE_GAS_URL` 留空或設為預設值，系統會自動使用內建的模擬資料，方便開發測試。

## 📖 使用說明

### 登入系統
- 輸入帳號和密碼登入
- 管理員帳號可以審核任務、查看收費統計、Excel 匯入收費資料
- 一般使用者可以操作任務、客戶等基本功能

### 任務管理流程
```
建立任務 → 處理中 → 勾選完成（自動記錄完成時間）→ 待審核 → 管理員審核 → 已審核
```

### SOP 套用流程
```
建立 SOP 流程 → 選擇 SOP + 客戶 → 自動建立包含所有步驟的工作任務
```

### Excel 批次匯入
1. 準備好 Excel 檔案（第一列為表頭，欄位名稱需與工作表一致）
2. 點擊各模組的「Excel 匯入」按鈕
3. 選擇檔案後預覽匯入資料
4. 確認後批次匯入

### 客戶分配
1. 新增客戶資料（不指定承辦人）
2. 在「客戶分配」頁面，待分配客戶會自動出現在左側
3. 點擊「分配」→ 從下拉選單選擇承辦人
4. 分配完成後，客戶會歸入該承辦人名下

## 🏗️ 系統架構

```
┌──────────────────┐     fetch API      ┌──────────────────┐     SpreadsheetApp    ┌──────────────────┐
│                  │ ──────────────────→ │                  │ ──────────────────→   │                  │
│   React (Vite)   │                    │  Google Apps     │                       │  Google Sheets   │
│   前端應用程式    │ ←────────────────── │  Script Web App  │ ←──────────────────   │  資料庫           │
│                  │     JSON response  │                  │     Read/Write        │                  │
└──────────────────┘                    └──────────────────┘                       └──────────────────┘
```

| 項目 | 技術 |
|------|------|

## ❓ 常見問題

**Q: 修改 Apps Script 後沒有生效？**
A: 每次修改程式碼都需要重新部署。點擊「部署」→「管理部署作業」→ 編輯 → 版本選「建立新版本」→ 部署。

**Q: 出現 CORS 錯誤？**
A: 確認 Apps Script 部署設定中「存取權限」設為「所有人」。

**Q: 離開開發模式沒有資料？**
A: 確認 `.env` 中的 `VITE_GAS_URL` 已正確設定為你的 Web App URL。

**Q: 如何新增員工帳號？**
A: 在「群組管理」頁面點擊「新增員工」，填寫帳號和密碼即可。或直接在 Google Sheets 的「群組管理」工作表新增。
