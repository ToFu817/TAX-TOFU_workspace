# 🧾 律岩案件管理系統 (AccLaw Case Management System)

這是一個專為會計師事務所設計的現代化案件管理系統，採用 **React + Google Apps Script (GAS) + Google Sheets** 架構。系統具備高度的靈活性，所有資料皆同步儲存在您的 Google 雲端硬碟中。

---

## ✨ 核心功能特色

### 📊 個人化儀表板
- **專屬統計**：無論管理員或員工，儀表板僅顯示「與自己相關」的案件狀態（待處理、延遲中等）。
- **緊急任務清單**：自動抓取「延遲中」或「7天內即將到期」的任務，支援內部捲軸滑動，不破壞版面。
- **年度任務摘要**：根據當前月份自動顯示該月應注意的會計年度工作。

### 📋 三層級權限控管
- **管理者 (Admin)**：擁有最高權限，可審核案件、分配客戶、管理員工帳號、查看統計資料。
- **資深使用者 (Senior User)**：可檢視「所有員工」的案件，但不可進行審核或客戶分配，僅能操作自己的案件。
- **一般使用者 (General User)**：權限最低，僅能查看與操作「指派給自己」的案件與客戶。

### 📅 任務管理與審核
- **狀態追蹤**：從建立任務、處理中、完成、待審核到已審核，完整生命週期追蹤。
- **智慧排序**：
  - 「待審核」分頁自動將**完成日期較舊**（等待最久）的案件排在最上方。
  - 「已審核」分頁自動將**審核日期較新**的案件排在最上方。
- **日期區間查詢**：支援依據審核日期或完成日期進行區間篩選。

### 📥 Excel 批次作業
- 全系統模組（客戶、任務、項目）皆支援 Excel (.xlsx/.xls/.csv) 批次匯入。
- 支援帶有「公式結果」的 Excel 檔案匯入。

## 🚀 部署與設定

### 1. 後端 (Google Apps Script)
1. 在 Google Sheets 建立新試算表。
2. 進入「擴充功能」->「Apps Script」，貼上 `google-apps-script/Code.gs`。
3. 執行 `initializeSheets` 函式進行初始化。
4. **部署為網頁應用程式**：設定執行身分為「我」，存取權限為「所有人」。
5. **重要**：每次修改後端後，請務必「建立新版本」部署，API URL 才會生效。

### 2. 前端 (GitHub Pages)
1. 本系統使用 **HashRouter** (井字號路由)，解決 GitHub Pages 重新整理 404 的問題。
2. 專案內含 `public/404.html` 自動跳轉腳本，提供雙重保險防止頁面丟失。
3. **環境變數**：請在專案設定中加入 `VITE_GAS_URL` 指向您的 GAS Web App URL。

## 📄 Excel 匯入格式參考 (標題列)

匯入時，Excel 的第一列標題必須與下方文字完全一致：

**【工作任務】**
> 任務編號、任務項目、客戶編號、公司行號名稱、承辦、預計完成日、實際完成日、狀態、SOP步驟、備註、審核人、審核時間

**【客戶資料】**
> 客戶編號、組織型態、公司行號名稱、承辦、目前狀態、年度註記、統一編號、稅藉編號、負責人、聯絡人、公司電話、聯絡手機、Email、備註、待分配
