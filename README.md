# 🧾 稅務小豆腐案件管理系統 (TAX TOFU Case Management System)

這是一個專為會計師事務所設計的現代化案件管理系統，採用 **React + Google Apps Script (GAS) + Google Sheets** 架構。本系統將 Google Sheets 當作資料庫，具備零成本、高度安全且資料完全私有的優點。

---

## 🚀 從零開始：系統建置全攻略

### 第一階段：準備資料庫 (Google Sheets)

1.  **建立雲端表格**：前往 [Google Sheets](https://sheets.google.com/) 建立空白試算表。
2.  **開啟腳本編輯器**：選單 **「擴充功能」 > 「Apps Script」**。
3.  **貼上後端程式碼**：將 `google-apps-script/Code.gs` 內容全部複製並貼上。
4.  **初始化工作表**：選擇函式 **`initializeSheets`** 並執行。

### 第二階段：資安與部署 (重要！)

為了確保資料安全，本系統啟用了 API Key 驗證機制：

1.  **設定指令碼屬性 (後端密碼)**：
    *   在 Apps Script 編輯器左側點選 **「專案設定」(齒輪圖示)**。
    *   捲動到最下方「指令碼屬性」，點擊 **「新增指令碼屬性」**。
    *   **屬性**：`WEB_API_KEY`
    *   **值**：設定您的自訂密鑰 (例如：`TaxToFu-secret-817`)。
2.  **部署為網頁應用程式**：
    *   點擊 **「部署」 > 「新增部署作業」**。
    *   選擇 **「網頁應用程式」**。
    *   **執行身分**：我。
    *   **誰可以存取**：所有人 (因為我們已有 API Key 驗證)。
    *   **複製 Web App URL**：取得結尾為 `/exec` 的網址。

### 第三階段：前端環境設定 (GitHub)

1.  **設定 GitHub Secrets**：
    *   進入 Repository 的 **Settings > Secrets and variables > Actions**。
    *   新增以下兩個 Secret：
        *   `VITE_GAS_URL`: 您的 GAS 部署網址。
        *   `VITE_API_KEY`: 剛才在 GAS 設定的 `WEB_API_KEY` 同一個值。
2.  **本機開發環境**：
    *   複製 `.env.example` 為 `.env`。
    *   填入相同的 `VITE_GAS_URL` 與 `VITE_API_KEY`。
    *   執行 `npm install` 與 `npm run dev`。

## ✨ 系統特色與優化

*   **⚡ 效能優化 (3個月過濾)**：
    *   任務管理頁面預設僅抓取「最近 3 個月」的資料，大幅提升載入速度。
    *   若需查看歷史資料，可點選工具列的 **「載入全部資料」**。
*   **🔒 權限與安全**：
    *   **管理權限**：僅「管理者」角色具備「審核」任務的權限，一般與資深使用者會看到權限提示。
    *   **API 加密**：所有與後端的通訊皆經過 API Key 校驗，防止介面被外部直接呼叫。
*   **📂 客戶管理**：
    *   **自動排序**：所有客戶清單預設按「客戶編號」由小到大排序。
    *   **承辦同步**：在客戶資料直接修改「承辦人員」，系統會自動同步至「客戶分配」分頁。

## 🛠️ 系統維護須知

### 1. 修改後端程式碼後
每次修改 `Code.gs` 或變更金鑰，**必須重新部署**：
*   點擊「部署」 > 「管理部署作業」。
*   點擊「編輯（鉛筆）」 > 版本選「建立新版本」 > 點擊「部署」。

### 2. 金鑰變更
若要修改 API Key，請務必同步更新以下三個地方：
1.  Google Apps Script 的指令碼屬性 (`WEB_API_KEY`)。
2.  GitHub Repository 的 Secrets (`VITE_API_KEY`)。
3.  本機開發環境的 `.env` 檔案。

## ❓ 常見問題解決

*   **Q: 為什麼網頁顯示「連線失敗」或「身份驗證失敗」？**
    *   A: 請檢查您的 `VITE_API_KEY` (前端) 與 `WEB_API_KEY` (後端屬性) 是否完全一致，並確認 GAS 已重新部署至新版本。
*   **Q: 任務管理頁面看不到舊資料？**
    *   A: 系統預設隱藏 3 個月前的資料以維持流暢度，請點擊「載入全部資料」按鈕即可看到。
